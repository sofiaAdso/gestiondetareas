🚀 PRÓXIMOS PASOS PARA TI
═══════════════════════════════════════════════════════════════════════════════

PASO 1: VERIFICAR BASE DE DATOS (2 minutos)
─────────────────────────────────────────────
Abre pgAdmin o psql y ejecuta:

    SELECT * FROM actividades LIMIT 1;
    SELECT id, nombre, email FROM usuarios;

Si la tabla NO existe:
    Ejecuta: crear_tabla_actividades.sql

Si NO hay usuarios:
    Ve a: http://localhost:8080/SistemaGestionTareas/registro_usuario.jsp

PASO 2: COMPILAR Y DESPLEGAR (3 minutos)
─────────────────────────────────────────────
Opción A - Con IDE:
    1. Abre el proyecto en IntelliJ IDEA
    2. Click en Run/Deploy
    3. Espera que compile

Opción B - Manual:
    1. Doble click en: compilar_desplegar.bat
    2. O ejecuta: mvn clean package
    3. Copia el WAR a Tomcat/webapps/
    4. Reinicia Tomcat

PASO 3: EJECUTAR DIAGNÓSTICO (1 minuto)
─────────────────────────────────────────────
Abre tu navegador y ve a:

    http://localhost:8080/SistemaGestionTareas/DiagnosticoServlet

Este servlet te dirá si todo está OK o qué falta.

PASO 4: CREAR TU PRIMERA ACTIVIDAD (2 minutos)
─────────────────────────────────────────────
1. Ve a: http://localhost:8080/SistemaGestionTareas
2. Inicia sesión
3. Click en "Gestionar Actividades"
4. Click en "Nueva Actividad"
5. Llena el formulario:
   • Título: "Mi Primera Actividad" (obligatorio)
   • Fecha Inicio: Hoy (obligatorio)
   • Fecha Fin: Dentro de 30 días (obligatorio)
   • Otros campos: Valores por defecto
6. Click en "Crear Actividad"

📊 FLUJO DE CREACIÓN DE ACTIVIDAD
═══════════════════════════════════════════════════════════════════════════════

Usuario llena formulario
         ↓
Validación JavaScript (cliente)
         ↓
[SUBMIT] POST a ActividadServlet
         ↓
Servlet valida parámetros
         ↓
Crea objeto Actividad
         ↓
Llama a ActividadDao.crear()
         ↓
Ejecuta INSERT INTO actividades
         ↓
PostgreSQL guarda el registro
         ↓
Retorna éxito/error
         ↓
Redirect a lista con mensaje

En cada paso hay LOGGING para que puedas ver exactamente dónde está el problema.

🆘 PRUEBA RÁPIDA SQL
═══════════════════════════════════════════════════════════════════════════════

Si quieres verificar que la BD funciona, ejecuta esto en psql/pgAdmin:

-- Ver usuarios
SELECT id, nombre, email FROM usuarios;

-- Crear actividad de prueba (usa un ID de usuario real)
INSERT INTO actividades
(usuario_id, titulo, descripcion, fecha_inicio, fecha_fin, prioridad, estado, color)
VALUES
(1, 'TEST', 'Prueba', CURRENT_DATE, CURRENT_DATE + 30, 'Media', 'En Progreso', '#3498db')
RETURNING id;

-- Verificar
SELECT * FROM actividades ORDER BY id DESC LIMIT 1;

-- Limpiar
DELETE FROM actividades WHERE titulo = 'TEST';

Si este SQL funciona → El problema está en la aplicación Java
Si este SQL falla → El problema está en la base de datos
